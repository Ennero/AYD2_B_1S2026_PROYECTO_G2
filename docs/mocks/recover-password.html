POST /api/auth/login

Entrada al sistema

Recibe `EMAIL` y `PASSWORD`, valida contra `USERS` y devuelve el token del portal correcto.

POST /api/auth/recovery

Paso 1: enviar token

Inserta `TOKEN_HASH`, `EXPIRES_AT` y `USER_ID` en `PASSWORD_RECOVERY_TOKENS`.

POST /api/auth/password

Paso 2: nueva contraseña

Marca `USED_AT` y actualiza `USERS.PASSWORD_HASH` cuando el token ya fue validado.
